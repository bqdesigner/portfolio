Talk · IA no processo de Product Design

Vamos falar de IA no processo de Product Design.


Não como hype, mas como ferramenta que amplifica nossa atuação.

Bruno Queirós · Stone · Senior Product Designer · 40min + 20min Q&A

Tese central

IA não substitui o designer.
Potencializa nossa atuação.

Quem tem repertório + contexto + processo, multiplica.
Quem não tem, gera slop mais rápido.

O que é slop?

Conteúdo gerado por IA com aparência de qualidade, mas sem fundamento, contexto ou direção.

Texto inflado, design genérico, decisão sem evidência. Volume sem valor.

03 / Mindset antes da ferramenta

4 princípios que vêm antes do prompt

5min
01

Contexto > prompt bonito

Bom input carrega contexto, insumos, não-ambiguidades. Prompt bonito sem contexto = output bonito sem fundamento.

02

Iteração > one-shot

Primeiro output é matéria-prima, não entregável. Refine em ciclos curtos.

03

Validar sempre

IA confiante ≠ IA correta. Você é o filtro. Sem isso, escala erro.

04

Taste é insubstituível

Julgamento estético, contexto político, leitura de stakeholder — IA não faz.

Onde a IA falha

Não delegue isto

—
01

Decisão de prioridade

Trade-off político e estratégico precisa de leitura humana.

02

Sensibilidade cultural

Contexto local, tom, código social — IA generaliza.

03

Trade-off com poucos dados

Sem evidências, IA preenche com o que ela considera bom no contexto — não com verdadeiro.

04

Coerência sistêmica longa

Janela de contexto se perde dentro da própria sessão.

4.1 / Discovery & Research

Sintetizar o que antes levava dias

3min

Ferramenta

Claude Code Gemini Perplexity

O que dá pra fazer

  • Sintetizar entrevistas
  • Desk research e benchmark competitivo
  • Clusterizar pain points
  • Analisar dashboards de dados
Antes

Tempo para encontrar players, aprofundar no contexto, coletar informações, montar board, criar análise.

Depois

Informe o contexto › informe o player › Tipo de análise › Onde montar (ex: FigJam) › Iterar e refinar.

−80% tempo
4.2 / Ideação & Estratégia

Do markdown ao Figma sem fricção

3min

Ferramenta

Claude Skills Figma MCP Obsidian

Truques

  • Documente em Markdown — Claude lê rápido + menos tokens
  • Base de conhecimento: transcrições + Jira/Confluence + ideações
  • Fluxo: Markdown → Mermaid → FigJam → Figma com Design System
Antes

Estruturar doc, desenhar fluxograma à mão, atrasar exploração visual.

Depois

Resultado prático cedo, validação rápida, foco em stakeholder, tempo para refinar visual.

−90% tempo
4.3 / Copy

Validação de tom direto no Figma

3min

Ferramenta

Claude Code Skills Figma

Jobs

  • Validação de content
  • Criação de FAQ
  • Verificação de tom de voz

Passo link do Figma → IA valida → grava output direto na tela → faço o check.

Antes

Verificar guia, alinhar com content, considerações de stakeholders, ciclos longos.

Depois

Chego na content com revisão prévia já feita pela IA.

−80% tempo
4.4 / Protótipo & Código

Protótipo com fidelidade de produção

3min

Ferramenta

Figma Make Claude Code

Jobs

  • Testar protótipo com inputs de dados reais
  • Maior fidelidade pro desenvolvimento
  • Validação tech precisa

Princípio

Designer entende o código que entrega.

Depois

Criar como seria em produção. Validação com tech mais precisa, menos retrabalho.

4.5 / Handoff, Doc & Automação

Terminal = IA com contexto real

3min

Ferramenta

Terminal Claude Code MCPs

Jobs

  • Specs automáticas
  • Changelog

06 / Como aprofundar amanhã

3 ações pro time essa semana

3min
01

Migrar 1 fluxo de research

Mover um fluxo real pra workflow Claude com contexto persistente. Medir tempo antes/depois.

02

Experimentar Cursor em 1 componente

Pegar um componente real e prototipar em código. Sentir a diferença de fidelidade.

03

Criar 1 skill / MCP custom

Automatizar uma rotina chata do seu próprio workflow. Compartilhar com o time.
